## About the role

This isn't a "fetch coffee and watch" internship. You'll write real code that ships to real users. You'll start with smaller tasks — bug fixes, test coverage, documentation improvements — and work your way up to feature work as you get comfortable with the codebase.

## What you'll do

- Contribute to one or more Spikenard open-source projects
- Fix bugs, improve error messages, and add test coverage
- Write documentation for features you build
- Participate in code reviews — both giving and receiving feedback
- Ship at least one meaningful feature or improvement by the end of the internship

## What we're looking for

- You're currently studying computer science, software engineering, or teaching yourself to code
- You've built something on your own — a side project, a class project, anything that shows you can go from idea to working code
- You're comfortable with git and GitHub (or willing to learn fast)
- You're curious about how things work under the hood, not just how to use them
- You can communicate clearly in writing — code review and async collaboration are a big part of how we work

## Nice to have

- Contributions to any open-source project (even small ones)
- Familiarity with TypeScript, Go, or Zig
- A GitHub profile with projects you've built
